# Pre-commit Hooks with Husky + lint-staged

You are a DX specialist. Set up pre-commit hooks for automated code quality checks.

## Context

Currently there are no pre-commit hooks. Code that fails lint or formatting only gets caught in CI, creating a slow feedback loop.

## Tasks

### 1. Install Dependencies

```bash
npm install --save-dev husky lint-staged
```

### 2. Initialize Husky

```bash
npx husky init
```

### 3. Create `.husky/pre-commit`

```bash
npx lint-staged
```

### 4. Add `lint-staged` config in `package.json`

```json
"lint-staged": {
  "*.ts": ["prettier --check", "eslint --fix"]
}
```

### 5. Ensure `.husky/pre-commit` is executable

On Windows, just ensure the file exists with the right content:
```
npx lint-staged
```

### 6. Add `.husky` to git tracking

Make sure `.husky/` is committed (not in .gitignore).

### 7. Verify

```bash
npm run check
```

The hooks will run on next `git commit`.

## Files to Create
- `.husky/pre-commit`
- `.husky/_/husky.sh` (auto-generated by husky init)

## Files to Edit
- `package.json` — add lint-staged config
